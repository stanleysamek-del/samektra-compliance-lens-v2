### Mobile + UI fixes

- **Bottom tab bar** — split into a 5-item `MOBILE_NAV` (Profile moved to header avatar). Previously 6 items in `grid-cols-4` were wrapping to a second row and breaking the raised Upload button placement.
- **Global `overflow-x: clip`** on `html, body` — catches any oversized descendant from causing horizontal page-scroll. Cheap safety net.
- **iOS input zoom fix** — `.cl-input` font-size 14px → 16px on mobile (drops back to 14px at sm:). Stops iOS Safari from force-zooming on focus, which was indirectly causing "menus positioned wrong" symptoms.

### Speed work

## How to enable / verify each feature

### AI_TWO_STAGE (still off — biggest perceived speed win still pending)

Vercel → Settings → Environment Variables → add `AI_TWO_STAGE = 1` for Production. Redeploy. Compare durations on `/admin/stats`. Roll back by setting to `0` (no code change needed).

### Why analyses sometimes take 40s instead of 8s

If a photo shows model `gpt-4o` and duration ~40s, the Anthropic fallback chain failed and the request fell through to OpenAI (slow on this prompt). Check Vercel runtime logs for `[ai] anthropic failed, trying next provider: ...` to see why — usually a bad/missing `ANTHROPIC_API_KEY`.

### Chip's rules

1. Open any inspection → photo → Coach Chip on a missed finding.
2. Tap "🧠 Teach Chip this" below Chip's reply, edit, Save.
3. Verify at `/team/rules` (Active list).
4. Upload a new photo → check that the rule's `times_applied` counter ticks up.
