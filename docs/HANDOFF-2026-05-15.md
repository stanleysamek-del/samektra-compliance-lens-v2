### 2. Transfer admin role

### 3. Two-stage analyzer (detect → focused analyze)

Adds a fast Haiku-only detection pass (~1s, ~$0.0005) that returns a list of compliance-relevant categories visible in the photo. The categories are then prepended to the analyze prompt as a focus hint, telling the model which rule blocks to apply. Speed gains come from shorter, more focused output (token generation is the slow part).

**Shipped DARK behind env flag.** Set `AI_TWO_STAGE=1` in Vercel to turn on. Per-request override available via `?two_stage=1` on `/api/analyze`.

**Files:**
- `lib/prompts/compliance-detect.ts` (new) — detection prompt, 20-category enum, focus-hint formatter
- `lib/ai/client.ts` — `detectCategories()`, `analyzeImageTwoStage()`, extended `analyzeImage()` with optional `focusCategories` arg
- `app/api/analyze/route.ts` — honors `AI_TWO_STAGE` env or `?two_stage=` URL param
- `app/api/photos/upload/route.ts` — honors `AI_TWO_STAGE` env, sums detect + analyze tokens in cost ledger

### 4. max_tokens cap

Dropped from 4096 → 2048 across Anthropic, OpenAI, Gemini providers. Real outputs are 500–1500 tokens; this caps worst-case runaway responses without changing normal-case speed.

### 5. Client-side image resize (verified already in place)

`lib/resize-image.ts` already caps at 1024px on the longest edge with 0.85 JPEG quality. No change needed.

## How to enable / test two-stage

After this branch deploys:

1. Vercel → Project → Settings → Environment Variables → add `AI_TWO_STAGE = 1` (Production)
2. Redeploy (or wait for next deploy)
3. Compare timings on `/admin/stats` for the same photo before/after toggling
4. Rollback: set `AI_TWO_STAGE = 0` and redeploy. No code change required.
